# Mozc Dictionary Tools (Download + Yomi→Surface Pair Builder)

Two small Python scripts:

1. **`10_download_mozc_dicts.py`**  
   Downloads Mozc OSS dictionaries (`dictionary00.txt` … `dictionary09.txt`) into `data/mozc/`.

2. **`20_build_pairs_from_mozc.py`**  
   Builds `yomi<TAB>surface` pairs from Mozc dictionaries and **generates all combinations** of variants for:
   - 濁点 → 清音（例: が→か）
   - 半濁点 → 清音（例: ぱ→は）
   - 小書き → 通常サイズ（例: っ→つ、ゃ→や）

   It can also filter by yomi length (min/max), deduplicate outputs, and limit combinatorial explosion.

---

## Requirements

- Python **3.8+** (recommended: 3.10+)
- No external dependencies (stdlib only)

## 1) Download Mozc Dictionaries

### Script
`scripts/10_download_mozc_dicts.py`

### Default behavior
Downloads:

- `https://raw.githubusercontent.com/google/mozc/master/src/data/dictionary_oss/dictionary00.txt`
- ...
- `dictionary09.txt`

into `data/mozc/`.

### Usage

```bash
python3 scripts/10_download_mozc_dicts.py
````

### Options

- Change output directory:

```bash
python3 scripts/10_download_mozc_dicts.py --out_dir data/mozc
```

- Change base URL:

```bash
python3 scripts/10_download_mozc_dicts.py --base_url https://raw.githubusercontent.com/google/mozc/master/src/data/dictionary_oss
```

- Overwrite existing files:

```bash
python3 scripts/10_download_mozc_dicts.py --force
```

## 2) Build `yomi<TAB>surface` Pairs (with Variant Expansion)

### Script

`scripts/20_build_pairs_from_mozc.py`

### Output format

Each line:

```
<yomi_variant>\t<surface>
```

Example:

- Original yomi: `がっこう`
- Variant positions: `が` (dakuten), `っ` (small)
- Generated: `がっこう`, `がつこう`, `かっこう`, `かつこう` (4 variants)

### Important behavior

- The script **only keeps entries whose first character of `yomi` is Hiragana** (same as your previous behavior).
- Variants are generated by scanning **all characters** in yomi.

## Typical Workflow

### Step 1: Download dictionaries

```bash
python3 scripts/10_download_mozc_dicts.py
```

### Step 2: Build pairs

```bash
python3 scripts/20_build_pairs_from_mozc.py --mozc_dir data/mozc --out_tsv out/mozc_pairs.tsv --dedup --only_changed
```

---

## Options (Pair Builder)

### Basic I/O

- `--mozc_dir`: directory containing `dictionary*.txt` (default: `data/mozc`)
- `--out_tsv`: output TSV path (default: `out/mozc_pairs.tsv`)

### Deduplication / limits

- `--dedup`: deduplicate lines globally
- `--max_lines N`: stop after writing N lines (0 = unlimited)

### Filtering for verification

- `--filter_yomi "<exact>"`: only output entries with exact yomi match
- `--filter_prefix "<prefix>"`: only output entries whose yomi starts with prefix

Example:

```bash
python3 scripts/20_build_pairs_from_mozc.py --filter_yomi がっこう --max_lines 50
```

### Output only “changed” entries

- `--only_changed`: only output entries where at least one variant was possible (i.e., yomi contains dakuten/handakuten/small)

### Prevent combinatorial explosion

- `--max_variants_per_entry N`:
  If variants for a single entry would exceed N, the entry is skipped.

Example:

```bash
python3 scripts/20_build_pairs_from_mozc.py --max_variants_per_entry 32
```

### NEW: Filter by yomi length

- `--min_yomi_len N`: minimum yomi length (characters). 0 = no minimum
- `--max_yomi_len N`: maximum yomi length (characters). 0 = no maximum

Examples:

- 3〜10文字のみ:

```bash
python3 scripts/20_build_pairs_from_mozc.py --dedup --only_changed --min_yomi_len 3 --max_yomi_len 10
```

- 6文字以下のみ:

```bash
python3 scripts/20_build_pairs_from_mozc.py --dedup --only_changed --max_yomi_len 6
```

- 8文字以上のみ:

```bash
python3 scripts/20_build_pairs_from_mozc.py --dedup --only_changed --min_yomi_len 8
```

> Note: length filtering is based on the **original yomi** (`len(e.yomi)`), not each generated variant.

## Output Notes

- Output encoding: UTF-8
- Newlines: `\n`
- If `--dedup` is enabled, dedup is performed on the exact output line (`"<yomi>\t<surface>"`).
